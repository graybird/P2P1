NeuroGrid Simulator Applet Instructions

The NeuroGrid Simulator applet can simulate a variety of routing methodologies in small peer to peer networks.  The code can be found at http://www.sourceforge.net/projects/neurogrid/ and includes classes that operate without the applet GUI that allow simulations to be performed with much larger numbers of nodes, and appropriate statistics to be calculated.

However the applet itself is a useful debugging tool to see what is going on in the network at a small scale.  Various parameters can be adjusted using the applet GUI, and hitting the restart button will stop the current network simulation and re-initialize the network using the new parameters.  

The basic operation of the simulation is to create a collection of documents that are then assigned keywords either randomly or according to the Zipf distribution.  Then the network is created according to the relevant parameters and documents distributed over the nodes.  Searches are then performed in serial, a document from the collection being selected at random and a search being conducted for that document by looking for nodes that contain documents with the keywords belonging to that document.  The GUI doesn't display much information about where which documents are located at the moment, but that is partly because I haven't had time to add that, and partly because it is not clear how best to display that information without hopelessly cluttering up the display.

The green lines are connections between nodes.  The squares are nodes, and the orange blobs are messages.  The nodes turn from white to gray when they receive a message allowing us to follow what proportion of the network has been reached in the current search.

You can see some of the results of simulations using this code at http://www.neurogrid.net/si-simulation02.html

The parameters affect the simulation as follows:

Zipf Checkbox When ticked the keywords are assigned to documents according to a Zipf Distribution, when not ticked the distribution if Uniform.
Knowledge Checkbox When ticked, nodes exchange lists of keywords that they have documents associated with, with their immediate neighbours.
Learning Checkbox When ticked the results of searches lead to the updating of knowledge base of the node initiating the search, i.e. it learns to associate the node that contained the relevant document with the keywords used in the search.
Ring Checkbox When ticked the nodes are connected in a regular fashion creating a ring.  When not ticked, the connections are created randomly.
Node Number The number of nodes the network starts with, and the number that will be added at the end of the Run Length
Thread Sleep The number of milliseconds pause in-between each message forward.  Going below 100 milliseconds in a browser normally crashes the system.  If you want more speed, download the code and run in an appletviewer.
Forwarding The number of connected nodes to forward a message to.  When set to zero each forwarding of a message will be to all connected nodes - Gnutella-like behaviour.
Run Length The number of searches performed before the Network grows by adding additional nodes equal to the Node Number
No Documents The number of documents in the simulation.
No Keywords The number of keywords in the simulation.
Keywords per Document The number of keywords assigned to each document.
Documents per Node The number of documents distributed to each node.
Start TTL The Time To Live (TTL) of each message, i.e. the number of times it can be forwarded before it expires.
Degree of Correlation Can be used to force initial neighbours to have related collections of documents.  The higher the value the more keywords that any two connected nodes will have in common at the start of the simulation - should really be replaced with an appropriate node addition scheme where nodes connect to other nodes with similar content.
Conns per Node The number of starting connections for each node.
Debug Verbosity Controls the verbosity of the simulation, i.e. the detail of the debug messages.
